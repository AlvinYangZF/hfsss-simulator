# Firmware Coverage Enhancement Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development for each test addition and superpowers:verification-before-completion before claiming coverage gains.

**Goal:** Raise firmware merged line coverage from 80.0% to at least 85.0% using UT/system/stress coverage paths.

**Status:** Completed on 2026-05-16. Final merged line coverage is 85.8%.

**Architecture:** Keep production firmware logic unchanged unless a test exposes a real defect. Add focused tests to existing binaries already present in `make test` and `COVERAGE_UT_BINS`, so the coverage pipeline does not need new build targets.

**Tech Stack:** C tests built by `Makefile`, lcov/genhtml coverage, QEMU E2E coverage through `scripts/coverage/run_e2e_coverage.sh`.
